# Plan: Add mouse scroll wheel support to labwc Alt+Tab window switcher

## Goal
Allow users to cycle through windows using the mouse scroll wheel while Alt+Tab (cycle mode) is active. This is a common feature in other compositors and window managers.

## Background
labwc's window switcher (`src/cycle/`) currently only supports keyboard navigation (Tab, arrow keys). Mouse scroll events during cycle mode are ignored — they fall through to `process_cursor_axis()` which processes mouse bindings and forwards to clients.

## Implementation

### Step 1: Create fork and feature branch
- Fork `labwc/labwc` on GitHub via `gh repo fork`
- Create branch `scroll-wheel-cycle` from master

### Step 2: Patch `src/input/cursor.c`
**File**: `src/input/cursor.c`
**Location**: Inside `handle_axis()` (line 1432), after `cursor_set_visible(seat, true)` (line 1441) and before the `scroll_factor` computation (line 1444).

Insert:
```c
/* Cycle mode: scroll wheel cycles through windows */
if (server.input_mode == LAB_INPUT_STATE_CYCLE) {
    if (event->orientation == WL_POINTER_AXIS_VERTICAL_SCROLL
            && event->delta != 0) {
        cycle_step(event->delta < 0
            ? LAB_CYCLE_DIR_FORWARD
            : LAB_CYCLE_DIR_BACKWARD);
    }
    return;
}
```

**Why this location**:
- Early interception before mouse bindings or client forwarding
- `cursor.c` already includes `cycle.h` (line 24) and `labwc.h` (line 31) — no new includes needed
- Follows existing pattern of cycle checks at lines 997, 1066, 1233
- `cycle_step()` is the same function called by keyboard handler (`handle_cycle_view_key()` in keyboard.c:487-493)

**Scroll direction mapping**:
- Scroll down (positive delta) → `LAB_CYCLE_DIR_FORWARD` (matches keyboard Down/Right)
- Scroll up (negative delta) → `LAB_CYCLE_DIR_BACKWARD` (matches keyboard Up/Left)
